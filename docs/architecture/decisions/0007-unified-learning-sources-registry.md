# ADR 0007: Unified learning-sources registry (reference-only)

Date: 2026-07
Status: Accepted

## Context

The product wanted to surface all 19 active Fleet project study queues,
research-paper paths, private Reader saves, the High Signal feed, and 12
embedded GitHub learning repositories — without copying canonical source
bodies into a separate editorial system. Copying would create a second
source of truth that drifts.

## Decision

Build a **reference-only** registry (`src/data/learning-sources.json`,
generated by `scripts/sync-learning-sources.mjs`) that indexes sources by
metadata and fetches bodies at request time through authenticated server
proxies. The `/sources` index, source detail, and `/session/:date/:sessionId`
routes are owner-only. The High Signal adapter consumes a versioned compact
daily feed, rejects unsupported payloads, and fails stale instead of
inventing content. The Reader adapter loads saved articles through the
authenticated server proxy — article bodies and Reader credentials are never
emitted into the static catalog or client bundle.

## Alternatives considered

- **Copy source bodies into the catalog.** Rejected: drift, licensing risk,
  and credential leakage for private sources.
- **Per-source bespoke integrations.** Rejected: a single registry + adapter
  contract is maintainable; bespoke code per source is not.

## Consequences

- External items can be studied in sessions, marked for FSRS review, saved
  into the notes store, and opened as a prefilled Playground exercise —
  without changing native concepts.
- A failed or empty upstream parse retains the previous generated source
  instead of silently dropping it (stale-on-failure contract).
- Activating the private Reader adapter requires setting
  `READER_API_TOKEN` — see
  [`../../operations/runbooks/reader-adapter.md`](../../operations/runbooks/reader-adapter.md).
